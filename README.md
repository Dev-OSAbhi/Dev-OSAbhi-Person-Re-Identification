# ICPR 2026 TVRID - Depth Track Solution

This repository contains the solution code for the **Depth Track** of the ICPR 2026 Competition on Privacy-Preserving Person Re-Identification from Top-View RGB-Depth Camera (TVRID).

## 1. Depth Track Approach: Depth Colorization with OSNet
Because standard Convolutional Neural Networks (CNNs) are highly optimized for natural images (RGB), they can struggle to extract meaningful hierarchical features from raw, single-channel depth maps. To overcome this, our solution adopts a **Depth Colorization** strategy combined with an **Omni-Scale Network (OSNet)** [1].

### Pipeline Overview
1. **Preprocessing (JET Colormap):** We transform the single-channel raw depth maps into 3-channel pseudo-color images using the `JET` colormap via OpenCV. This transformation assigns a spectrum of colors to different depth values (e.g., blue for near, red for far), effectively encoding pure geometric distance into visual texture and edge patterns.
2. **Backbone (OSNet):** The colorized depth maps are fed into an `osnet_x1_0` backbone. The Omni-Scale Network is particularly powerful for top-view re-identification because its unified aggregation gate successfully handles the severe perspective distortion and scale variations present in overhead camera data.
3. **Initialization:** By converting the depth data into a 3-channel "visual-like" representation, we initialized the OSNet backbone with **ImageNet pre-trained weights**. This leverages powerful transfer learning, providing a vastly superior starting point for convergence compared to random initialization on raw depth arrays.
4. **Distance Metric:** Inference outputs high-dimensional embeddings that are L2-normalized. The final rankings are generated by computing the Euclidean distance across embeddings.

### Training Configuration
* **Optimizer:** Adam Optimizer
* **Learning Rate:** 0.003
* **LR Scheduler:** MultiStepLR (Drop at epoch 30 and 50)
* **Max Epochs:** 60
* **Batch Size:** 32
* **Loss Function:** Softmax classification loss (Cross-Entropy with Label Smoothing)
* **Augmentation:** Random Crop and Horizontal Flip

## 3. 📂 Dataset Setup

Follow the steps below to correctly set up the dataset for this project.

### 1️⃣ Create the Data Folder
Create a folder named **`data`** in the **root directory** of the project.

### 2️⃣ Download the Dataset
Download the dataset from the link below:

🔗 **[Download Dataset](https://zenodo.org/records/17909410)**

⚠️ **Important:**  
Make sure to download **`extracted.zip`** and **NOT** `original.zip`.

### 3️⃣ Extract the Dataset
After downloading:
- Extract **`extracted.zip`**
- Rename / keep the folder as **`DB_extracted`**
- Place it inside the **`data`** folder so the structure looks exactly like this:

```
data/DB_extracted/
├── train_labels.csv
├── public_test_labels.csv
├── train/
└── test_public/
```

## 4. Run the Code

### Step 1: Train the Model (Depth Track)
Execute the following command to train the `osnet_x1_0` model using the custom `TVRID` dataloader which applies the Jet Colorization strategy dynamically on the depth images.

*(Note: Ensure GPU 1 is available, or change `--gpu` to your target device id like `--gpu 0`)*

```bash
python train_tvrid.py \
  --root data/DB_extracted \
  --track depth \
  --save-dir log/tvrid_depth \
  --gpu 1
```
This will run the training loop for 60 epochs and save checkpoints in the `log/tvrid_depth` directory.

### Step 2: Generate Submission Rankings
Once training finishes, you generate the `rankings_depth.csv` submission file by pointing the `make_submission.py` script to your final `.pth.tar` checkpoint model. 

```bash
python make_submission.py \
  --root data/DB_extracted \
  --csv data/DB_extracted/public_test_labels.csv \
  --track depth \
  --checkpoint log/tvrid_depth/model.pth.tar-60 \
  --output rankings_depth.csv \
  --gpu 1
```

*Important:* If your training stops before epoch 60, make sure to update `--checkpoint log/tvrid_depth/model.pth.tar-60` to match the exact checkpoint file that was successfully generated.
